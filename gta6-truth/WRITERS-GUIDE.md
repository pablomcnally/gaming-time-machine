# Writer Guide

This site is a static parody news site. Writers should edit the story data, then rebuild the generated pages.

## Online Editing

Open the editor at:

```text
/editor/
```

Use the shared editor password.

When you click **Save to GitHub**, the editor commits the story data to GitHub. Vercel should then redeploy the live site automatically.

## Tip Inbox

Reader tips from the homepage tip form appear in the editor's **Tip inbox**.

Use **Make draft** to turn a tip into a starter story. It will add a new draft story at the top of the story list.

After you have saved the story, use **Dismiss** to remove the tip from the inbox.

Tips are stored in:

```text
data/tips.json
```

## The Important Files

If you are editing locally instead of using the online editor, edit this file:

```text
data/stories.json
```

Do not manually edit files inside:

```text
stories/
```

Those article pages are generated by the build script and will be overwritten.

## Adding A New Story

Open `data/stories.json` and add a new object inside the `stories` array.

Use this template:

```json
{
  "slug": "example-story-url",
  "category": "leak",
  "categoryLabel": "Leak",
  "badge": "Unverified",
  "accent": "blue",
  "title": "Your funny headline goes here.",
  "description": "One sharp sentence that sums up the joke and appears on the homepage and in search previews.",
  "imageUrl": "",
  "imageAlt": "",
  "author": "Writer Name",
  "readTime": "3 min read",
  "date": "June 9, 2026",
  "body": [
    "First paragraph of the story.",
    "Second paragraph of the story.",
    "Third paragraph of the story."
  ]
}
```

## Field Notes

`slug` becomes the URL. Keep it lowercase, short, and use hyphens:

```text
stories/example-story-url/
```

`category` controls the homepage filter. Use one of:

```text
leak
analysis
local
```

`categoryLabel` is what readers see. Examples:

```text
Leak
Analysis
Local chaos
```

`badge` appears on the coloured story image block. Keep it short:

```text
Unverified
Deep dive
Local chaos
Tip line
Enhanced 800%
```

`accent` controls the colour of the story block. Use one of:

```text
blue
yellow
green
red
sunset
```

`description` should be one sentence. Think of it as the homepage standfirst and Google preview.

`imageUrl` is optional. Leave it blank to use the coloured story block. Add a full image URL or a local asset path to show an image:

```text
https://example.com/image.jpg
assets/story-images/example.jpg
```

`imageAlt` should briefly describe the image. Leave it blank if the image is decorative.

In the online editor, the easier route is:

```text
Choose file -> Upload and use image -> Save to GitHub
```

Uploaded images are stored in:

```text
assets/story-images/
```

Use JPG, PNG, WebP, or GIF. Keep images under 3 MB.

`body` is an array of paragraphs. Put each paragraph in its own quoted line.

## Rebuilding The Site

If you edited stories locally, run this from the `gta6-truth` folder:

```powershell
npm.cmd run build
```

That regenerates:

```text
index.html
stories/<story-slug>/index.html
stories/stories.json
```

## Publishing Changes

If you edited locally, publish from the main repo folder:

```powershell
git add gta6-truth
git commit -m "Add new GTA parody story"
git push origin main
```

Vercel should deploy automatically after the push.

## Writing Style

The site works best when stories sound weirdly confident about obviously flimsy evidence.

Good headline shape:

```text
Trailer reflection may confirm entire fishing economy, claim experts with zoom tools.
```

Good story tone:

```text
Keep the voice newsy, over-serious, and slightly ridiculous. The joke should be that the article treats tiny rumours like major public-interest journalism.
```

Avoid:

```text
Official-sounding claims that look real
Real private people
Real legal accusations
Official GTA or Rockstar branding
Long paragraphs
```

Always keep the parody disclaimer in place.
